# AgentsForge 系统构建规划

## 核心理念

**Model is Agent**，我们要做的只是给 Model 创造 harness 环境，核心包括：

1. 实现工具
2. 策划知识
3. 管理上下文
4. 控制权限
5. 多 Agent 协作机制
6. 收集过程数据
---

## 构建阶段

| 阶段 | 内容 | 核心产出 |
|------|------|----------|
| P1 | 循环 + 工具 + 规划 | 有规划能力的单 Agent 诞生 |
| P2 | 记忆 + 持久化 | 具备上下文管理，持久化记忆功能 |
| P3 | 并发 + 多 Agent 团队 | 多 Agent 角色协作，效率与效果同时升华 |
| P4 | 自治 + 隔离 | 协作机制升级，环境隔离，任务自领 |
| P5 | 真实复杂 benchmark 测试 | 复杂环境真实测试，问题暴露与解决 |
| P6 | 确定过程数据格式，制定进化算法规则 | 难以想象的大工程 |

ps:根据之前My_Agents项目的构建，其中发现了很多问题，主要是项目架构设计的不合理应发的洪灾，所以现在重新构建更加强大的AgentsForge系统。
