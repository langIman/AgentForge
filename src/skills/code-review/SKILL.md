# Code Review — 代码审查技能

## 触发条件
当用户要求审查代码、进行code review、或检查代码质量时加载此技能。

## 审查流程

1. **理解上下文**：先阅读相关文件，理解代码的目的和架构
2. **逐文件审查**：按以下维度检查每个文件
3. **生成报告**：汇总问题并给出改进建议

## 审查维度

### 正确性
- 逻辑错误、边界条件、空值处理
- 类型安全、异常处理

### 安全性
- 注入风险（SQL/命令/XSS）
- 敏感信息泄露
- 权限检查

### 可维护性
- 命名清晰度
- 函数长度（建议<50行）
- 重复代码

### 性能
- 不必要的循环/查询
- 内存泄漏风险
- 异步阻塞

## 输出格式

```
## Code Review Report

### 🔴 Critical (必须修复)
- [文件:行号] 描述

### 🟡 Warning (建议修复)
- [文件:行号] 描述

### 🟢 Good (做得好的地方)
- 描述

### 建议
- 改进建议列表
```
