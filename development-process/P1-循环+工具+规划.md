## 逐模块说明

### 1. 核心骨架 (`src/core/`)

#### config.py — 配置中心

所有可调参数集中在这一个文件里，通过 `.env` 文件或环境变量覆盖：

| 配置项 | 默认值 | 作用 |
|--------|--------|------|
| `MODEL_NAME` | claude-sonnet-4-20250514 | 使用的LLM模型 |
| `WORKSPACE_ROOT` | ./workspace | 文件操作的沙箱根目录 |
| `BASH_TIMEOUT` | 120秒 | Shell命令超时 |
| `TODO_MAX_ITEMS` | 20 | 待办列表上限 |
| `TODO_NAG_INTERVAL` | 3 | 每N轮提醒更新待办 |
| `DANGEROUS_COMMANDS` | rm -rf /, mkfs等 | 危险命令黑名单 |

#### state.py — Agent状态

```python
class AgentState(TypedDict):
    messages: list[AnyMessage]    # 对话历史（LangGraph自动合并）
    session_id: str               # 会话ID
    todos: list[TodoItem]         # 待办列表
    rounds_since_todo: int        # 距离上次更新todo的轮次
```

后续Phase只需**新增字段**，已有字段永不修改。

#### graph.py — 不可变图拓扑

这是整个系统的骨架，**永远不改这个文件**：

```
用户输入 → pre_process → agent(LLM) → 有tool_calls? ─是→ tools → post_process → 回到pre_process
                                            │
                                           否 → END（返回给用户）
```

所有定制都通过注入不同的 `nodes` 和 `tools` 来实现，图本身不变。

#### nodes.py — 节点逻辑

| 节点 | 职责 |
|------|------|
| `pre_process` | P1阶段仅透传，P2会加入压缩逻辑 |
| `agent` | 拼接system prompt + 历史消息，调用LLM |
| `should_continue` | 检查LLM回复中是否有tool_calls，决定走tools还是END |
| `post_process` | **Nag机制**：超过3轮没更新todo且有未完成项，注入提醒消息 |

### 2. 工具层 (`src/tools/`)

#### bash.py — Shell执行（s01-s02）

- 在 `WORKSPACE_ROOT` 目录下执行shell命令
- **危险命令过滤**：匹配黑名单（rm -rf /、mkfs等）→ 直接拦截
- **超时保护**：默认120秒，超时自动终止
- 输出截断至50000字符，防止上下文爆炸

#### file_ops.py — 文件三件套（s02）

三个工具共享一个**路径沙箱**机制：

```
_safe_path(file_path)
  → 解析为绝对路径
  → 检查是否在 WORKSPACE_ROOT 内
  → 越界则拒绝
```

| 工具 | 功能 |
|------|------|
| `read_file` | 读文件，支持行号偏移和行数限制，输出带行号 |
| `write_file` | 写文件（覆盖），自动创建父目录 |
| `edit_file` | 精确字符串替换，要求old_string在文件中唯一 |

#### todo.py — 待办管理（s03）

`TodoManager` 类管理全局待办列表，**两个硬约束**：

1. **最多20条** — 防止Agent无限拆分任务
2. **同时仅1个 in_progress** — 强制Agent专注做一件事

渲染格式：
```
[x] 已完成的任务
[>] 正在进行的任务
[ ] 待办任务
```

配合 `nodes.py` 中的 **Nag机制**：如果Agent连续3轮不更新todo，post_process会注入一条提醒消息，迫使Agent汇报进度。

#### subagent.py — 子Agent（s04）

解决的问题：主Agent探索代码库时，中间过程会撑大上下文。

方案：`spawn_subagent` 工具启动一个**独立子图**：
- 独立State，不共享主Agent的消息历史
- 独立工具集（仅 read_file + bash）
- **无checkpointer**，不持久化
- 最多10轮交互
- 只返回最终AI消息的前5000字符

效果：主Agent的上下文只增加一条tool_result（摘要），而不是子Agent的全部中间过程。

#### skill.py — 技能系统（s05）

**两层注入**设计，平衡 token 消耗和信息完整性：

| 层 | 时机 | 内容 | Token消耗 |
|----|------|------|----------|
| Layer 1 | 每轮（system prompt） | 技能名 + 一行描述 | ~100 tok/技能 |
| Layer 2 | 按需（tool调用） | 完整SKILL.md | ~2000 tok |

Agent先从Layer 1知道有哪些技能可用，需要时再通过 `load_skill` 加载完整内容。

### 4. CLI入口 (`src/cli.py`)

一个简单的REPL循环：

```
启动 → 创建Agent图 → 等待用户输入 → invoke图 → 提取AI回复 → 打印 → 循环
```

特殊命令：
- `todos` — 打印当前待办列表
- `quit` / `exit` — 退出
- `Ctrl+C` — 中断当前执行

## 关键设计决策

### 为什么图拓扑永不修改？

所有Agent（主Agent、子Agent、未来的Worker）共用同一个图结构。差异化通过注入不同的 `nodes` 和 `tools` 实现。这保证了：
- 行为可预测（都是同一个循环）
- 调试简单（只需关注节点内部逻辑）
- 后续Phase扩展不会破坏已有功能

### 为什么Todo有两个硬约束？

- **20条上限**：防止Agent过度拆分任务，浪费token在todo管理上
- **单in_progress**：防止Agent同时推进多个任务导致上下文混乱，强制线性推进

### 为什么子Agent不共享checkpointer？

子Agent是一次性的探索任务，持久化没有意义。不共享checkpointer还能防止子Agent的中间状态污染主会话的恢复点。

### 为什么文件操作需要沙箱？

Agent可以执行任意文件操作。沙箱确保它只能在 `WORKSPACE_ROOT` 内读写，防止误删系统文件或读取敏感信息。bash工具则通过危险命令黑名单提供另一层保护。
